# host-pocket 真實使用者操作 SOP（房東 + 房客）

> **線上版：** [https://host-pocket.vercel.app](https://host-pocket.vercel.app)  
> **建議裝置：** 手機瀏覽器（Safari / Chrome），也可開桌面版  
> **建議語言：** 繁中（右上角可切 EN）  
> **更新：** 2026-07

---

## 使用前準備

- [ ] 開無痕視窗 A → 扮演「房東」
- [ ] 開無痕視窗 B → 扮演「房客」（模擬收到連結）
- [ ] 準備一組測試 Email（預定確認用，例：`yourname+test@gmail.com`）
- [ ] 示範房源 ID（四選一）：
  - `TAIPEI-CITY` — 台北信義
  - `UK-LONDON` — 倫敦
  - `VILNIUS-OLDTOWN` — 維爾紐斯
  - `RIO-COPACABANA` — 里約
- [ ] 或自訂 ID：任意新代碼（例 `MY-STAY-2026`）→ 系統會產生通用指南

## A. 房東流程（第一次建立指南）

### A-1 進入 App

1. 開啟 [https://host-pocket.vercel.app](https://host-pocket.vercel.app)
2. 確認語言為「繁中」
3. 應看到配對首頁（連接房源 / 快速開始）

### A-2 快速體驗（推薦新手）

1. 點 Tab「**快速開始**」
2. 點「台北 · 信義區天空盒」（或任一示範主題）
3. 系統自動配對 → 進入指南主頁
4. ✅ 檢查：有房源照片、Wi-Fi、4 張推薦卡片

### A-3 用自己的 Listing ID（進階）

1. 回到配對首頁 → Tab「**連接房源**」
2. 輸入 Airbnb `rooms/` 後數字，或自訂代碼 `MY-STAY-2026`
3. 按「**連結房源並建立指南**」
4. ✅ 新 ID 會有 fallback 內容，可再到設定頁客製

### A-4 編輯住宿指南（房東後台）

1. 配對頁底部 → 點「**編輯住宿指南**」  
   或直接開：  
   `https://host-pocket.vercel.app/host-settings.html?listing=TAIPEI-CITY`
2. 輸入相同 Listing ID → 按「**載入**」
3. 編輯：
   - **基本資訊：** Wi-Fi、電子鎖、房源照片
   - **推薦地點 1–4：** 標題、封面、距離、價格、推薦文案
   - 可點「**快速選擇景點**」帶入範本
4. 按「**儲存設定**」→ 看到成功提示
5. 可點「**預覽 App**」確認房客畫面
6. 回 App，用**相同 ID** 重新配對 → 確認內容已更新

### A-5 取得分享連結

1. 在指南主頁，點房源照片區「**分享**」
2. 連結已複製到剪貼簿
3. 預期格式類似：
   - `https://host-pocket.vercel.app/guide/TAIPEI-CITY`
   - `https://host-pocket.vercel.app/index.html?listing=TAIPEI-CITY`
4. 把連結貼到記事本，待會給「房客視窗 B」使用

## B. 房客流程（假裝剛 check-in 收到連結）

### B-1 打開指南

1. 在視窗 B 貼上 A-5 的分享連結
2. ✅ 直接進指南主頁（不需安裝 App）
3. ✅ 看到房東名稱、地點、入住天數等資訊

### B-2 入住須知 Tab

1. 確認 Wi-Fi 名稱／密碼可讀
2. 點電子鎖區塊 → 查看解鎖說明
3. 可點「取得動態密碼」走示範流程
4. ✅ 資訊足夠讓房客自行連線、開門

### B-3 在地體驗 Tab

1. 切到「**在地體驗**」
2. 左右滑動 4 張推薦卡片
3. ✅ 每張有：標題、標籤、距離、價格、評分（無評分顯示 `—` 屬正常）
4. 點任一卡片 → 開啟體驗詳情
5. ✅ 詳情含：照片/影片、評價、取消政策、「**預定體驗**」按鈕

### B-4 預定體驗（核心路徑）

1. 詳情頁按「**預定體驗**」
2. 日曆：應顯示**當月真實日期**，今天有橘框
3. 點選一個可預約日期（過去日期不可選）
4. 右側選時段（半小時一格；彈性時段顯示「當日時段皆可預約」）
5. 時區維持 `Asia/Taipei`（或改「其他」自填）
6. 按確認 → 輸入測試 Email
7. 按「**確認預定**」
8. 預期結果（二選一）：
   - 已設定 SMTP/Resend → 收到確認信
   - 未設定 → 開啟郵件 App，內文已帶預定資訊（mailto 備援）
9. ✅ 確認信內容含：體驗名稱、日期、時段、地點

### B-5 實境探索

1. 底部導覽 →「**實境探索**」
2. ✅ 地圖有起點／終點 pin 與虛線
3. 切換推薦 1 / 2 → 路線文字會更新

### B-6 求助房東

1. 底部 →「**求助房東**」
2. ✅ 可「傳送訊息給房東」（開 mailto）
3. ✅ 可快速跳轉 Wi-Fi、電子鎖、推薦區

### B-7 語言切換

1. 頂部切「EN」
2. ✅ 標題、Tab、按鈕改英文
3. 切回「繁中」

## D. 常見狀況對照（排查用）

| 現象 | 可能原因 | 怎麼確認 |
|------|----------|----------|
| 評分顯示 `—` | 設定頁未填 `recRating` | 正常；補 `4.9` 即可 |
| 看到 `NaN` | 舊版 bug | 重新整理；應已修復 |
| 預定只開 mailto | 後端未設 SMTP | 查 `/api/health` 的 `bookingConfigured` |
| 設定存了 App 沒變 | Listing ID 不一致 | 兩邊 ID 必須完全相同 |
| 日曆沒可選日期 | 該月無符合時段 | 換下個月或換示範房源 |
| 體驗詳情載入失敗 | SearchAPI 未設 | mock 模式用本地 fixture |

**健康檢查：** [https://host-pocket.vercel.app/api/health](https://host-pocket.vercel.app/api/health)  
→ 查看 `dbConfigured` / `bookingConfigured` 了解後端狀態

## 附錄：本機開發測試

若要在本機跑完整流程（含設定 API、寄信）：

```bash
npm install
cp .env.example .env   # 填入 DATABASE_URL、Resend 或 SMTP
npm start
```

- App：`http://localhost:3000`
- 設定頁：`http://localhost:3000/host-settings.html?listing=TAIPEI-CITY`
- 寄信測試（管理者）：`http://localhost:3000/email-test.html`
